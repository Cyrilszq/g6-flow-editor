提供一个基于 G6 3.0 实现流程图编辑的示例，预览地址：http://g6.demo.cyrilszq.cn

```
npm install
npm run start
```

### 想法
不打算在 G6 的基础上进行封装，因为总会遇到各种各样的特殊业务逻辑，封装后的接口很难满足业务需求，通常都还要进行定制。
所以希望以自定义节点、自定义行为的形式来对 G6 进行一些增强，暂时只考虑流程图编排的情况。

已有功能：
- 边的选中行为
- 将节点拖拽到画布行为
- 连接两个锚点行为（配合带锚点的自定义节点 —— anchor-node）
